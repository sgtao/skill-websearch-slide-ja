Claude Code 実践ガイド

Claude Code に MCPサーバーを導入する

外部ツール・DB・APIを Claude Code から直接使う ― HTTP / stdio / OAuth、スコープ、チーム共有まで

公式docs準拠 claude mcp add HTTP / stdio / OAuth
出典: Claude Code 公式ドキュメント（code.claude.com/docs/en/mcp）。コマンド仕様はバージョンにより変わる場合があります。
MCPとは？ ― 何ができて、何が要るか

MCP（Model Context Protocol）は、AIと外部ツールをつなぐオープン標準。接続すると、Claude Code が課題管理・監視・DB などを直接読み書きできる。

  • 別ツールからチャットにコピペしている作業があれば、それが接続のサイン
  • 前提：Claude Code が動作していること＋追加先サーバー（URL もしくは実行コマンド）
  • 接続方式は3種：HTTP（推奨）／stdio（ローカル）／SSE（非推奨）
STEP 1
接続方式とスコープを決める
  • 方式：リモートは http 推奨、ローカルプロセスは stdio、sse は非推奨
  • スコープ3種：local（自分＋現プロジェクト・既定）／project（.mcp.jsonで共有）／user（全プロジェクト）優先順位は Local > Project > User
  • 書式ルール：--transport / --env / --scope / --header はサーバー名の前。stdio は -- でコマンドを区切る
# 基本の形
claude mcp add [--transport ...] [--scope ...] <name> <url または -- command>
STEP 2
HTTPサーバーを追加する（推奨）
# 基本形（リモートHTTPサーバー）
claude mcp add --transport http <name> <url>

# 例: Notion を追加
claude mcp add --transport http notion https://mcp.notion.com/mcp

# 認証ヘッダー付き
claude mcp add --transport http secure-api https://api.example.com/mcp \
  --header "Authorization: Bearer your-token"

リモート接続は HTTP が最も安定。URL と（必要なら）認証ヘッダーを渡すだけ。

STEP 3
ローカル（stdio）サーバーを追加する
# 基本形（-- の後に実行コマンドと引数）
claude mcp add --transport stdio <name> -- <command> [args...]

# 例: airtable（環境変数つき・npx 実行）
claude mcp add --transport stdio airtable \
  --env AIRTABLE_API_KEY=YOUR_KEY \
  -- npx -y airtable-mcp-server

-- より前がオプション、後ろが「サーバーを起動するコマンド」。秘密情報は --env で渡す。

STEP 4
認証して有効化する（OAuth）

多くのクラウド型サーバーは OAuth 認証が必要。追加してからセッション内で /mcp を実行して認証する。

# OAuthが必要なサーバーを追加
claude mcp add --transport http sentry https://mcp.sentry.dev/mcp

# Claude Code セッション内で認証
> /mcp

トークンは安全に保存され、自動更新される。/mcp メニューから管理・取り消しが可能。

STEP 5
確認・管理して、チームで共有する
# 設定済みサーバーと接続状態を一覧
claude mcp list

# プロジェクト共有（.mcp.json に保存され、リポジトリで共有）
claude mcp add --transport http --scope project team-api https://api.example.com/mcp
  • 設定変更後は claude mcp list で状態確認。セッション内は /mcp で接続・ツールを管理
  • .mcp.json は環境変数展開に対応：${VAR} / ${VAR:-default}（url・headers・env 等で有効）
つまずきポイント & Tips
  • node / npx が見つからない：サブプロセスのシェル環境が端末と異なるため絶対パスを使うか、~/.zshrc 等で nvm を読み込む（~/.zprofile だけでは不足しがち）
  • Windows の stdio：npx は cmd /c で包む例：claude mcp add --transport stdio my -- cmd /c npx -y @some/package
  • 迷ったら HTTP：SSE より安定し、公式も推奨
  • バージョン差に注意：新しめの版は add-json 形式、古い版は add --transport 形式が必要な場合がある
まとめ
  • 接続方式を選ぶ（リモート=HTTP推奨／ローカル=stdio／SSEは非推奨）
  • claude mcp add で追加 ― オプションはサーバー名の前、stdioは -- で区切る
  • クラウド型は /mcp で OAuth 認証、claude mcp list で状態確認
  • --scope project＋.mcp.json（環境変数展開対応）でチーム共有
