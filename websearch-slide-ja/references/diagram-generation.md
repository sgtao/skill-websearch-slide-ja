# diagram-generation.md — インラインSVG図解の生成

構造・流れ・関係・循環を **外部依存ゼロのインラインSVG** で描く。
Step 3-0（図解採用判定）と Step 4（図解埋め込み）で読む。

## いつ使うか（グラフ・画像との使い分け）

| 伝えたいもの | 手段 |
|---|---|
| 数量比較・推移・割合 | グラフ（chart-generation.md） |
| **構造・流れ・関係・循環** | **図解（本ファイル）** |
| 実物・外部の権威ある図 | 画像（image-embedding.md） |

- パターン選択：手順・処理の流れ → **横フロー**／構成・階層 → **縦レイヤー**／循環 → **サイクル**。

## 厳守ルール（グラフと同じ思想）

- **外部依存ゼロ**：SVG図形のみ。`<image>` や外部 `href` / `xlink:href` 禁止（CDN不要・PDF/PNG完全対応）。
- **色のハードコード禁止**：`fill="#…"` を書かない。必ず `diagram-*` クラス経由。アクセントは `--accent`。
- **正確性ルール**：図に描く要素・矢印は **出典で確認できた関係に限る**。推測のコンポーネント／接続を足さない。出典は `diagram-source`（svg内）か直後の `.note` に明記。
- **共存禁止**：グラフ・図解・画像を同一スライドに置かない。1スライド1ビジュアル。
- **ラベル折り返し**：SVG `<text>` は自動折り返ししない。長いラベルは `<tspan>` で **手動 2 行** に分割する（下記）。

### 長いラベルの2行分割（tspan）

```html
<text class="diagram-label" x="CX" y="CY">
  <tspan x="CX" dy="-0.6em">1行目</tspan>
  <tspan x="CX" dy="1.2em">2行目</tspan>
</text>
```

## 共通雛形

```html
<div class="slide-diagram">
  <svg viewBox="0 0 960 480" xmlns="http://www.w3.org/2000/svg" class="diagram-svg" role="img">
    <title>（図の要約・a11y用）</title>
    <defs>
      <marker id="arrow" viewBox="0 0 10 10" refX="9" refY="5"
              markerWidth="7" markerHeight="7" orient="auto-start-reverse">
        <path d="M0,0 L10,5 L0,10 z" class="diagram-arrowhead" />
      </marker>
    </defs>

    <!-- ノードとエッジ（パターン別座標表に従う） -->

    <text x="920" y="470" class="diagram-source" text-anchor="end">出典: …</text>
  </svg>
</div>
```

- スライド見出しは既存の `.slide-title` を使う（svg内にタイトルを重複させない）。
- エッジは `<path class="diagram-edge" marker-end="url(#arrow)" d="M x1,y1 L x2,y2"/>`。

## パターン1: 横フロー（手順・処理の流れ）N=2〜5

- ノード矩形：`y=185 height=90 rx=8`（縦中心 230）。ラベルは各ノード中心。
- 上限 N=5。超えるなら分割（前半／後半の2スライド等）。

| N | ノード幅 | ノード左x（中心x） | 矢印 x1→x2（y=230） |
|---|---|---|---|
| 2 | 420 | 40(250), 500(710) | 460→500 |
| 3 | 267 | 40(174), 347(480), 653(787) | 307→347, 614→654 |
| 4 | 190 | 40(135), 270(365), 500(595), 730(825) | 230→270, 460→500, 690→730 |
| 5 | 144 | 40(112), 224(296), 408(480), 592(664), 776(848) | 184→224, 368→408, 552→592, 736→776 |

例（N=3 の1ノード目）:
```html
<rect class="diagram-node" x="40" y="185" width="267" height="90" rx="8"/>
<text class="diagram-label" x="174" y="230">情報収集</text>
<path class="diagram-edge" marker-end="url(#arrow)" d="M307,230 L347,230"/>
```

## パターン2: 縦レイヤー（アーキ構成）L=2〜4層 × 各層 k=1〜4箱

- 層バンド：`x=40 幅=Wk`、層高 H、最上段 y=50。箱は層バンド一杯（`height=H rx=8`）。
- 層は原則「積み重ね」（矢印なし）。基盤層は k=1（全幅）で表現。

層の y / 高さ：

| L | 各層 y（高さ H） |
|---|---|
| 2 | 50, 245 （H=175） |
| 3 | 50, 180, 310 （H=110） |
| 4 | 50, 147, 245, 342 （H≈77） |

層内 横位置（箱左x）：

| k | 箱幅 | 箱左x |
|---|---|---|
| 1 | 880 | 40 |
| 2 | 428 | 40, 492 |
| 3 | 277 | 40, 341, 642 |
| 4 | 202 | 40, 266, 492, 718 |

- 箱中心 = 箱左x + 箱幅/2、層中心 = 層y + H/2。ラベルはその中心。
- 層間に方向を示すときのみ縦矢印：`d="M480,(上層下端) L480,(下層上端)"`。

## パターン3: サイクル（循環・好循環）N=3〜4

- 中心 (480,235)、ノード中心半径 R=150、ノード＝円 `r=58`。
- 中央に循環テーマを `diagram-label` で置ける。
- 矢印は隣接ノードへ **直線弦**（時計回り）。曲線弧は v1.1 送り。

ノード中心：

| N | ノード中心（時計回り ①②③④） |
|---|---|
| 3 | ①(480,85) ②(610,310) ③(350,310) |
| 4 | ①(480,85) ②(630,235) ③(480,385) ④(330,235) |

矢印（start → end。端は円周から離す）：

| N | 矢印 |
|---|---|
| 3 | ①→②(509,135→577,253) ②→③(552,310→416,310) ③→①(379,260→447,142) |
| 4 | ①→②(521,126→583,188) ②→③(589,276→527,338) ③→④(439,344→377,282) ④→①(371,194→433,132) |

例（ノード①）:
```html
<circle class="diagram-node" cx="480" cy="85" r="58"/>
<text class="diagram-label" x="480" y="85">投資</text>
<path class="diagram-edge" marker-end="url(#arrow)" d="M509,135 L577,253"/>
```
